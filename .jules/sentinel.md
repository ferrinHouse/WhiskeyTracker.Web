## 2025-02-14 - Fix SSRF and Path Traversal in Photo Uploads
**Vulnerability:**
1. Server-Side Request Forgery (SSRF): The `GooglePhotoUrl` hidden input field could be manipulated to force the server to issue HTTP GET requests to internal/private IP addresses or other unintended endpoints via `HttpClient.GetAsync()`.
2. Path Traversal: The user-uploaded file's name (`ImageUpload.FileName`) was concatenated directly into a file path without sanitization, allowing an attacker to specify relative paths (e.g. `../../../`) to overwrite files elsewhere on the filesystem.

**Learning:**
1. Client-provided URLs should never be trusted or queried unconditionally by the server. Even if a URL is generated by a legitimate frontend script (like the Google Photos picker), an attacker can easily submit arbitrary values directly.
2. Form file uploads present similar risks with metadata. The `FileName` property from an `IFormFile` could be crafted maliciously.

**Prevention:**
1. Validate client-supplied URLs by verifying they use a secure scheme (HTTPS) and their hostname matches a strict allowlist (e.g., `endsWith(".googleusercontent.com")`).
2. Always sanitize filenames provided by users via `Path.GetFileName()` to extract only the filename component and strip any directory traversal characters before appending it to a secure upload path.
